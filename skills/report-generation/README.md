# 研究报告生成 Skill

## 适用场景

当 Agent 需要把结构化候选数据和分析结论整理成可读报告时使用。

## 输入文件格式

- 结构化候选 JSON
- 标签建议 JSON
- 分析中间结果 JSON
- 项目说明或 notes Markdown

## 输出文件格式

优先：

- Markdown 报告

可选：

- `.docx`

建议结构：

1. 摘要
2. 样本范围
3. 主要发现
4. 证据样本
5. 代表性原文片段
6. 结论
7. 下一步建议

## 执行步骤

1. 先写样本边界。
2. 再写筛选规则。
3. 按路径机制组织正文，不按平台流水账组织。
4. 每个结论附 1 到 3 条脱敏证据。
5. 长原文只取短片段。
6. 标明哪些部分是观察，哪些是推断。

## 示例 Prompt

```text
请把当前候选数据整理成一份研究简报，重点写年份差异、阶段差异、代表路径机制和仍需人工复核的风险点。
```

## 注意事项

- 不暴露隐私信息。
- 不堆平台截图或长原文。
- 不把报告写成平台运营周报。
